Layouts and the Add Page, Page

Building the front page

The front page of WikiStack should list all of the pages we have in our app and also have a link to create a new page. For this, we will need a couple of things.

  • An index.html view
  • Define the new page action for our router
  • Define the index action in the router to find all the pages

Let's start by defining our home page at index.html view and also add an Add Page view.

Layouts

In most websites, many pages share code like menus, footers, etc and it's silly to keep repeating this code for every page that we make.

Swig has a great feature called "layouts" that will help us with that.

Create a file called views/layout.html in Sublime and let's get started. Make sure this file is in the views folder.

Building the Wiki Layout

The great thing about layouts is that they allow you to define sections of the page, called blocks. Blocks are parts of the code that other pages on your site can customize. For example, each new page will want to customize their page title. So let's start our layout.html by adding a block where the title is.

<!-- layout.html -->
<!DOCTYPE html>
<html lang="en">
  <head>
    <title>{% block title %}WikiStack{% endblock %}</title>
    <link href="http://netdna.bootstrapcdn.com/bootstrap/3.0.3/css/bootstrap.min.css" rel="stylesheet">
  </head>
  <body>
   {% block content %}{% endblock %}
  </body>
</html>

We have defined a very bare-bones page above with a css include and two blocks: one in the <title> tag called "title", and one in the body called "content". The text "WikiStack" inside the "title" block is the default text.

Next, let's see what would happen when we create another page in this site called about_us.html where we would want to use this layout without having to re-write all of this code.

1
2
3
4

<!-- about_us.html -->
{% extends "layout.html" %}
 
{% block title %}WikiStack - About Us{% endblock %}
 
{% block content %}
  <p>We are a cool startup that are making a Node.js wiki that supports Markdown!</p>
{% endblock %}

Notice the extends "layout.html" command at the top of the file. This means everything we already defined in layout.html doesn't need to be written again. All we have to do is define the blocks, and our whole page will be ready to go! Cool right?

Routes for a new page

Receiving a new page submit will mean adding a new GET route and a new POST route to the app.js. The GET route is to render the add form itself, and the POST route will be to receive the submit. Let's add those two routes in the JS.

To do this, we will have to add a Add routes module. Make a new file in the routes folder using the routes/users.js file as a template. You can use the following Terminal commands:

Add Route Actions

Once our routes are defined, we need to define the route actions.

Let's first do the root Add form page route first - that one should be easy! All this function should do is render the "add" view. Try writing out the "/" action in the routes/add.js file yourself before checking the hint below.

 Hint: Add Route Action

router.get('/', function(req, res) {
  res.render('add');
});

The code above will simply render the add.html template. Now if you go to http://localhost:3000/add, you should see your add page form! Cool.

Double checking the form

Make sure that your form's HTML includes an opening <form> tag that looks something like the following. The important part here is to make sure the action, /add/submit is consistent with your route and also the method is set to POST:

</form>

Once our form is setup correctly, we can start to work on the POST route action that will catch the form submit.

The POST Action

Let's start our POST route with this skeleton (remember, this is to be added to /routes/add.js):

router.post('/submit', function(req, res) {
  var models = require('../models/');
 
  // STUDENT ASSIGNMENT:
  // add definitions of the `title`, `body` and `url_name` variables here
 
  var p = new models.Page({ "title": title, "body":body, "url_name": url_name});
  p.save();
  res.redirect('/');
});

In the code snippet above, add definitions to the title, body and url_name variables and then add the code above to your add.js routes file.

By the url_name value, we mean a version of the "title" that has no non-alphanumeric characters and uses underscore instead of space. To get the url_name, you will need to transform the name using regexes and the String.replace function of JavaScript. This will be used in the urls of the pages, e.g. a page titled "Page About My Family" would have the url: http://localhost:3000/Page_About_My_Family

In case there is no title, or the title is empty, you can use a randomly generated string. We have done it for you in the HINT below. Don't look there if you want to try it yourself first!

 Hint: Solution for URL_NAME Generator

var generateUrlName = function(name) {
  if (typeof name != "undefined" && name !== "") {
    // Removes all non-alphanumeric characters from name
    // And make spaces underscore
    return name.replace(/[\s]/ig,"_").replace(/[^\w]/ig,"");
  } else {
    // Generates random 5 letter string
    return Math.random().toString(36).substring(2,7);
  }
};
 
var url_name = generateUrlName(title);
